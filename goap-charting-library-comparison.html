GOAP Diagram Library Comparison

Four single-file HTML prototypes using Feature 4, each with a compact header/current-step diagram and a full standalone review/retry flow.

review index

Recommendation

The header diagram and the standalone page diagram should be separate artifacts. The header version should be a single row that shows the current stage. The standalone version should show named reports, blackboard facts, critique-derived conditions, and the redo path.

Default docs choice: Mermaid

Best if the diagrams stay in source-controlled documentation and exact lane geometry is not required.

Best split-view prototype: JointJS

Best control over a short header strip plus a full lane diagram, because it is precise SVG authoring.
