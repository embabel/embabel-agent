# Growth & Visibility Guide

This document provides actionable strategies to increase Embabel Agent Framework's visibility and community growth.

## Project Positioning

Embabel occupies a unique position in the AI agent landscape:

- **The Spring MVC of AI Agents**: While Spring AI provides low-level LLM access (like the Servlet API), Embabel offers high-level abstractions for complex agent workflows
- **GOAP-Powered Planning**: Unlike FSM-based frameworks, Embabel uses Goal Oriented Action Planning for dynamic, adaptive decision-making
- **JVM-Native**: Built for Java/Kotlin developers who need enterprise-grade reliability with Spring Boot integration
- **From the Creator of Spring**: Strong credibility in the enterprise Java community

### Key Differentiators vs Competitors

| Feature | Embabel | Spring AI | Koog | Python Frameworks |
|---------|---------|-----------|------|-------------------|
| Planning Algorithm | GOAP + Utility AI | Manual | Basic | Varies |
| Type Safety | Strong | Strong | Strong | Weak |
| Spring Integration | Native | Native | Limited | None |
| Enterprise Ready | Yes | Yes | Yes | Varies |
| Dynamic Replanning | Yes | No | Limited | Limited |
